# 조건부 분기 (Conditional Edge)

---

## 왜 필요한가?

`add_edge`는 항상 같은 다음 노드로 고정 연결된다. 하지만 실제로는 **state 값에 따라 다음에 실행할 노드가 달라져야** 하는 경우가 많다.

- 예: 사용자 입력이 질문이면 `answer_node`, 명령이면 `command_node`로 분기
- 예: 검증 결과가 실패면 `retry_node`, 성공이면 `END`로 분기

이럴 때 쓰는 게 `add_conditional_edges` 이다.

---

## 기본 사용법

```python
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    number: int
    result: str

# 분기 전에 거치는 노드 (아무 작업 안 해도 됨 - 그냥 통과용 노드)
def check(state: State) -> State:
    return {}

# 라우팅 함수 : state를 보고 "다음에 갈 노드 이름(문자열)"을 반환
def route(state: State) -> str:
    if state["number"] % 2 == 0:
        return "even"
    return "odd"

def even_node(state: State) -> State:
    return {"result": f"{state['number']} 는 짝수"}

def odd_node(state: State) -> State:
    return {"result": f"{state['number']} 는 홀수"}

graph = StateGraph(State)
graph.add_node("check", check)
graph.add_node("even", even_node)
graph.add_node("odd", odd_node)

graph.add_edge(START, "check")

# check 노드 다음에 route() 함수의 반환값에 따라 분기
# route()가 "even"을 반환하면 -> even 노드로, "odd"를 반환하면 -> odd 노드로
graph.add_conditional_edges(
    "check",                          # 분기가 일어나는 기준 노드
    route,                            # 라우팅 함수 (state -> 문자열)
    {"even": "even", "odd": "odd"}    # 라우팅 함수 반환값 : 실제 노드 이름 매핑
)

graph.add_edge("even", END)
graph.add_edge("odd", END)

app = graph.compile()

print(app.invoke({"number": 4, "result": ""}))  # {'number': 4, 'result': '4 는 짝수'}
print(app.invoke({"number": 7, "result": ""}))  # {'number': 7, 'result': '7 는 홀수'}
```

---

## 포인트

- `add_conditional_edges(기준_노드, 라우팅_함수, 매핑_딕셔너리)`
  - 라우팅 함수는 **state를 받아서 문자열(다음 노드를 가리키는 key)을 반환**해야 함
  - 매핑 딕셔너리의 key는 라우팅 함수 반환값, value는 실제 노드 이름
  - 매핑 딕셔너리를 생략하면 라우팅 함수의 반환값을 그대로 노드 이름으로 사용함
- 조건부 분기의 목적지로 `END`를 직접 매핑할 수도 있음 (예: 검증 실패 시 종료)
- 여러 조건부 분기를 조합하면 **반복(loop)** 구조도 만들 수 있음 (예: 실패 시 이전 노드로 되돌아가기)
