# 이터레이터 (Iterator)

---

## 왜 필요한가?

```python
# 리스트: 100만개를 메모리에 전부 올린 뒤 순회
nums = [i for i in range(1_000_000)]  # 즉시 100만개 생성 → 메모리 낭비

# range: 이터레이터 기반 → 지금 필요한 숫자만 하나씩 계산
for i in range(1_000_000):  # 메모리에 숫자 하나씩만 올라감
    print(i)
```

## 이터레이터 프로토콜

파이썬에서 이터레이터가 되려면 두 가지 메서드가 필요하다.

- `__iter__()` : 이터레이터 객체 자신을 반환
- `__next__()` : 다음 값을 반환. 더 이상 값이 없으면 `StopIteration` 예외 발생

```python
class CountUp:
    """0부터 max-1까지 하나씩 반환하는 이터레이터"""

    def __init__(self, max):
        self.max = max
        self.current = 0  # 현재 위치를 기억하는 상태값

    def __iter__(self):
        # for 루프나 iter()가 호출할 때 이터레이터 객체를 반환
        # 자기 자신이 이터레이터이므로 self 반환
        return self

    def __next__(self):
        if self.current >= self.max:
            # 더 이상 반환할 값이 없으면 StopIteration 발생
            # for 루프는 이 예외를 받으면 자동으로 순회를 종료함
            raise StopIteration

        value = self.current
        self.current += 1  # 다음 호출을 위해 상태 업데이트
        return value


counter = CountUp(3)

print(next(counter))  # 0
print(next(counter))  # 1
print(next(counter))  # 2
print(next(counter))  # StopIteration 발생
```

## for 루프가 내부적으로 하는 일

```python
nums = [10, 20, 30]

for n in nums:
    print(n)

# 위 for 루프는 내부적으로 아래와 동일하게 동작한다
it = iter(nums)       # nums.__iter__() 호출 → 이터레이터 객체 반환
while True:
    try:
        n = next(it)  # it.__next__() 호출 → 다음 값 반환
        print(n)
    except StopIteration:
        break         # StopIteration 발생하면 루프 종료
```

## 이터러블 vs 이터레이터

헷갈리기 쉬운 개념이라 구분해야 한다.

```python
# 이터러블(Iterable): __iter__() 만 있으면 됨 → iter()로 이터레이터를 만들 수 있는 객체
# 예: list, tuple, str, dict, range

nums = [1, 2, 3]      # 이터러블
it = iter(nums)        # 이터레이터 (이터러블에서 만들어짐)

# 이터레이터(Iterator): __iter__() + __next__() 둘 다 있는 객체
# 이터레이터는 항상 이터러블이기도 하다 (자기 자신을 반환하는 __iter__ 포함)

print(iter(it) is it)  # True → 이터레이터의 __iter__는 자기 자신을 반환

# 리스트는 이터러블이지만 이터레이터는 아님
# → next(nums) 하면 에러 (리스트는 __next__가 없음)
try:
    next(nums)
except TypeError as e:
    print(e)  # 'list' object is not an iterator
```

## 이터레이터는 한 번만 순회 가능

```python
it = iter([1, 2, 3])

for n in it:
    print(n)  # 1, 2, 3

# 이미 끝까지 소진된 이터레이터는 다시 순회해도 아무것도 나오지 않음
for n in it:
    print(n)  # 아무것도 출력 안 됨

# 리스트는 이터러블이므로 iter()를 다시 호출할 수 있어서 여러 번 순회 가능
nums = [1, 2, 3]
for n in nums:
    print(n)  # 1, 2, 3
for n in nums:
    print(n)  # 1, 2, 3 (다시 순회 가능)
```

## 매직 메서드 (Magic Method)

`__iter__`, `__next__` 처럼 `__` 두 개로 감싸진 메서드를 **매직 메서드** 라고 한다.
던더 메서드(dunder method, double underscore의 줄임)라고도 부르며 같은 말이다.

**직접 호출하지 않는 게 파이썬 관례.** 내장 함수를 통해 간접 호출한다.

```python
nums = [1, 2, 3]

# 매직 메서드 직접 호출 - 가능하지만 관례상 쓰지 않음
nums.__iter__()
nums.__len__()

# 내장 함수로 호출 - 파이썬 관례
iter(nums)   # 내부적으로 nums.__iter__() 호출
len(nums)    # 내부적으로 nums.__len__() 호출
next(it)     # 내부적으로 it.__next__() 호출
```

자주 쓰이는 매직 메서드:

| 내장 함수 / 연산자 | 호출되는 매직 메서드 |
|--|--|
| `iter(x)` | `x.__iter__()` |
| `next(x)` | `x.__next__()` |
| `len(x)` | `x.__len__()` |
| `str(x)` | `x.__str__()` |
| `x + y` | `x.__add__(y)` |
| `x[0]` | `x.__getitem__(0)` |
| `for x in y` | `y.__iter__()` |

---

## 정리

```
이터러블   →  __iter__() 만 있음  →  리스트, 튜플, 문자열 등
이터레이터 →  __iter__() + __next__() 있음  →  iter() 로 만들거나 직접 구현
```

- 이터레이터는 **상태(현재 위치)를 기억**한다.
- 값을 **하나씩, 필요할 때** 꺼내므로 메모리 효율적이다.
- 한 번 소진되면 재사용 불가. 다시 쓰려면 새로 만들어야 한다.
- 클래스로 직접 구현하면 번거롭다 → 제너레이터로 해결.
